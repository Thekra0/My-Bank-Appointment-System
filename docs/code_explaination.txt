شرح طريقة عمل الكود

===== كيف يعمل النظام بشكل عام؟ =====

عندما يزور المستخدم الموقع، يحدث التالي:

1. يُحمّل ملف header.php الذي يتصل بقاعدة البيانات (config/db.php)
2. يتم التحقق من وجود جلسة نشطة (Session)
3. يعرض شريط التنقل (navbar.php) بناءً على حالة تسجيل الدخول
4. يعرض محتوى الصفحة المطلوبة
5. يُحمّل footer.php الذي يحتوي على مكتبات JavaScript

===== ملف config/db.php =====

هذا الملف هو نقطة الاتصال المركزية:

- يحدد معلومات الاتصال بقاعدة البيانات (localhost, root, اسم القاعدة)
- ينشئ اتصال mysqli بقاعدة البيانات
- يبدأ جلسة Session إذا لم تكن موجودة
- يوفر دوال مساعدة:
  * isLoggedIn(): للتحقق من تسجيل الدخول
  * isAdmin(): للتحقق من صلاحيات المدير
  * requireLogin(): لحماية الصفحات
  * cleanInput(): لتنظيف المدخلات
  * setMessage() و getMessage(): لإدارة الإشعارات

===== نظام تسجيل الدخول =====

auth/login.php:

1. يتحقق إذا كان المستخدم مسجل دخول مسبقاً → يُحوّل للرئيسية
2. عند إرسال النموذج:
   - ينظف البيانات المدخلة
   - يبحث عن البريد في قاعدة البيانات
   - يتحقق من كلمة المرور باستخدام password_verify()
   - إذا صحيحة: يخزن بيانات المستخدم في Session
   - يُحوّل المستخدم للصفحة الرئيسية

auth/register.php:

1. يتحقق من ملء جميع الحقول
2. يتأكد من تطابق كلمتي المرور
3. يتحقق من عدم وجود البريد مسبقاً
4. يشفر كلمة المرور بـ password_hash()
5. يُدرج المستخدم الجديد في جدول users
6. يُحوّل للصفحة تسجيل الدخول

===== إدارة المواعيد =====

pages/appointments.php:

الإضافة:
- يستقبل البيانات من النموذج (الخدمة، التاريخ، الوقت)
- يتحقق من صحة البيانات
- يدرج الموعد في جدول appointments بحالة 'upcoming'

التعديل:
- يستقبل معرف الموعد والبيانات الجديدة
- يتحقق أن الموعد يخص المستخدم الحالي
- يحدث البيانات باستخدام UPDATE

الإلغاء:
- يغير حالة الموعد (status) من 'upcoming' إلى 'cancelled'

الحذف:
- يحذف الموعد نهائياً من القاعدة باستخدام DELETE

العرض:
- يجلب جميع مواعيد المستخدم الحالي مرتبة حسب التاريخ
- يعرضها في جدول مع أزرار الإجراءات

الطباعة:
- تستخدم JavaScript لفتح نافذة جديدة
- تكتب HTML بسيط يحتوي على تفاصيل الموعد
- تستدعي window.print() للطباعة

===== نظام التقييمات =====

pages/rating.php:

1. يجلب المواعيد المكتملة التي لم يتم تقييمها:
   - استعلام LEFT JOIN بين appointments و ratings
   - يظهر فقط المواعيد بحالة 'completed' بدون تقييم

2. عند إضافة تقييم:
   - يتحقق أن الموعد مكتمل ويخص المستخدم
   - يتحقق من عدم وجود تقييم سابق
   - يدرج التقييم (1-5 نجوم) والتعليق في جدول ratings

3. يعرض تقييمات المستخدم السابقة
4. يعرض تقييمات جميع العملاء الآخرين

===== لوحة التحكم الإدارية =====

pages/dashboard.php:

1. تحمي الصفحة: requireLogin() + isAdmin()
2. تجلب الإحصائيات:
   - عدد المواعيد (قادمة، مكتملة، ملغاة)
   - عدد المستخدمين
   - متوسط التقييم

   - Bar Chart: للخدمات الأكثر طلباً

4. تعرض جدول أحدث المواعيد مع JOIN مع جدول users

index.php:

   - تعرض مواعيده القادمة

===== حماية الصفحات =====

كل صفحة داخلية (appointments, rating, dashboard) تستخدم:

requireLogin():
- تتحقق من وجود Session نشطة
- إذا لم توجد → تحول للصفحة login.php
- هذا يمنع الدخول المباشر للصفحات بدون تسجيل

isAdmin():
- تتحقق من أن role = 'admin'
- تُستخدم في dashboard.php فقط
- تحمي الصفحات الإدارية من المستخدمين العاديين

===== التعامل مع البيانات =====

جميع المدخلات تمر بـ cleanInput():
1. trim(): إزالة المسافات الزائدة
2. stripslashes(): إزالة الشرطات المضافة
3. htmlspecialchars(): تحويل الأحرف الخاصة
4. real_escape_string(): حماية من SQL Injection

جميع الاستعلامات تستخدم Prepared Statements:
```php
$stmt = $conn->prepare("SELECT * FROM users WHERE email = ?");
$stmt->bind_param('s', $email);
$stmt->execute();
$result = $stmt->get_result();
```

===== نظام الإشعارات =====

setMessage():
- تخزن الرسالة ونوعها في Session

getMessage():
- تقرأ الرسالة من Session
- تحذفها بعد القراءة (تُعرض مرة واحدة)

في header.php:
- إذا وجدت رسالة → تعرضها بـ SweetAlert2

===== تدفق طلب نموذجي =====

مثال: حجز موعد جديد

1. المستخدم يملأ النموذج في appointments.php
2. عند الإرسال: $_POST['add_appointment'] موجود
3. cleanInput() ينظف البيانات
4. التحقق من صحة التاريخ (في المستقبل)
5. Prepared Statement يُدرج الموعد
6. setMessage() تخزن رسالة نجاح
7. header('Location: ...') يُحوّل للصفحة نفسها
8. header.php يعرض الرسالة بـ SweetAlert
9. الصفحة تعرض الموعد الجديد في الجدول
